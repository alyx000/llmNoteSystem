# 好的 agent 工程师如何衡量

> 由 `/ingest` 从 log 心得（2026-05-02）升格而来。

## 定义

核心命题：能在 agent 加持下从 0 到 1 高效构建大型项目，**且不牺牲项目质量**。

「不牺牲质量」是关键约束——很多人能用 agent 快速 ship 一个原型，但能否在大型、长期维护的项目上保持质量是分水岭。

## 关键能力维度

- **深度投资自己的 agent 工具链**（见 [[agent-toolchain-investment]]）——把 hooks、slash commands、自定义 agent 当长期资产
- **对项目质量有可操作的度量方式**（见 [[agent-adversarial-quality-test]]）——用 agent 攻击系统作为评估手段
